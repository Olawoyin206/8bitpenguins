# 8bit Penguins - Dev Log

## Project Overview
NFT minting application for generating 8-bit pixel penguin NFTs with customizable traits or transforming uploaded PFPs into penguin style.

## Tech Stack
- React 19 + Vite 7
- Ethers.js 6 (wallet/MetaMask integration)
- Pinata (IPFS image storage)
- JSONBin (shared gallery sync)

- Address: `0xab5ffaa03F6A2713b854a3144086e746303b12a3`

## Issues & Fixes

### 2026-02-23: JSONBin Rate Limiting
**Problem**: JSON requests being made every 5 seconds even when not generating images, causing JSONBin credits to reduce rapidly.

**Root Cause**: `fetchFreshGallery()` is called every 5 seconds in a `setInterval`:
```javascript
const interval = setInterval(() => {
  fetchFreshGallery().then(gallery => setSharedGallery(gallery))
}, 5000)
```

**Fix**: Removed the 5-second polling. Gallery only fetches:
1. On initial load
2. After user generates/transforms a penguin

*Status: FIXED*

## 2026-04-02: Gallery Loading Regression Fix (Mint + Evolve)
**Problem Found**:
- Gallery rendering was blocked at page level while metadata hydration was still in-flight.
- Owned-token discovery did a full `ownerOf` scan before trying transfer logs, causing slow first load.
- A single slow metadata call could stall the batch with no timeout.

**Fixes Applied**:
- `src/Mint.jsx`
  - Removed full-grid blocking on pending per-card metadata.
  - Added metadata fetch timeout guard for gallery hydration batches.
- `src/Evolve.jsx`
  - Removed full-grid blocking on pending per-card metadata.
  - Added metadata fetch timeout guard for gallery hydration batches.
- `src/ownedTokenDiscovery.js`
  - Switched discovery flow to logs-first for cold loads.
  - Kept `ownerOf` scan as fallback only.

**Expected Result**:
- Galleries show cards immediately with per-card loading states.
- Wallet NFT discovery is significantly faster on first load.
- Slow RPC calls fail fast instead of stalling the whole gallery.

**Verification**:
- `npm run build` passed.
